Encoding Rules 4 维度同时编码，互不干扰

UBoE 100 GB/s~5.0 px

§6 带宽 → 粗细 映射

wForBw() · log10 scale 公式：max(1.5, log10(bw/10) × 3.8)
BW (GB/s) → stroke-width (px) · 对数尺度
实线 = 铜缆 (copper) · 短虚线 7+5 = 光纤 UB (optical) · 稀虚线 4+7 = 全光长距 (opticalLong)

§7 使用规则速查

Quick Reference

🚫不可混用

实线 ↔ 虚线介质区分，不可混
glow 用于密集模式性能问题，密集时关闭
casing 用于密集模式同上，密集时省略
color 随意指定必须来自 D.colors / D.parallels
opacity 自定义必须用 credOpacity(cred) 计算

✓分层规范

G.floor地面网格，始终在最底层
G.fabric背景暗示，第二层
G.nodes芯片/交换机色块
G.active所有流量激活连线
G.flow流动光点（animateMotion）
G.labels文字标注，始终最顶层
新增连线时的检查清单：
① 带宽已知 → 用 wForBw(bw) 计算粗细，不要手写 px 数
② 可信度已知 → 用 credOpacity(cred) 计算透明度
③ 介质已知 → copper=实线 · optical=dasharray:7 5 · opticalLong=dasharray:4 7
④ 节点数 ≤30 → sparse模式（casing + glow + 流动光点）· >30 → dense模式（pulse动画）
⑤ 浅色背景视图 → 使用 #vglow，不用 #glow / #sglow
⑥ 注解引线 → dasharray:3 2 · stroke-width:1 · opacity:.65~.7 · 颜色与层色相同
