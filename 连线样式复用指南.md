# 仿真调试视图 · 连线（总线）样式复用指南

> 把"仿真调试 3D 视图"里的彩色流动连线 + 分层走线 + 端点接点 + 分类显隐，
> 整套搬到其他项目。本文按"数据 → 走线 → 渲染 → 样式 → 交互"五层拆解，
> 每层给出**实际代码**和**移植要点**。
>
> 源文件：`web-app/src/sim/IsoCanvas.tsx`（渲染）、`serverData.ts`（数据）、`simStore.ts`（开关状态）。

---

## 0. 它能复用到什么场景

任何 **React Three Fiber（R3F）+ three.js** 的 3D 场景，只要你的"节点"有：
- 一个**世界坐标位置**（在哪）
- 一组**连接声明**（连到哪条线、是起点还是终点、从哪个面引出）

就能复用。和具体是不是"服务器主板"无关——换成网络拓扑、管线图、流程节点都成立。

---

## 1. 整体架构（5 层）

```
① 数据层    BUS_REGISTRY（线种类） + 每个组件的 busConnections（连接声明）
                         │  getComponentsOnBus(busId) 把"谁在这条线上"聚合出来
                         ▼
② 走线层    computeRoute3()  master→slave，抬到分层高度，画 X→Z 直角折线
                         │  roundedCurve3() 给折角加圆角
                         ▼
③ 几何层    TubeGeometry 沿曲线挤出圆管 + 两端 connector 接点
                         ▼
④ 样式层    ShaderMaterial（DASH_VERT/FRAG）实色管体 + 流动"彗星"高亮
                         │  BUS_COLORS 每类一个颜色
                         ▼
⑤ 交互层    逐根 hover 变实 / 选中高亮 / 分类显隐开关（simStore.hiddenBusTypes）
```

## 3. ① 数据层 —— 你唯一需要"适配"的部分

### 3.1 三个类型 + 颜色表（直接抄）

```ts
export type BusType = 'POWER' | 'I2C' | 'PCIE' | 'SATA' | 'USB';   // 改成你的线种类

export interface BusConnection {
  busId: string;                                  // 挂在哪条总线
  busType: BusType;
  role: 'master' | 'slave' | 'peer';              // master=走线起点，slave=终点
  connectorPos: 'top' | 'bottom' | 'left' | 'right'; // 从节点哪个面引出
}

export interface BusDef {
  id: string; type: BusType; label: string;
  bandwidth: string; frequency: string;
  busStatus: 'active' | 'idle' | 'error';
}

// 每类线一个颜色 —— 图例和 3D 线共用这一份，保证永远一致
export const BUS_COLORS: Record<BusType, { hex: number; css: string; width: number }> = {
  POWER: { hex: 0xFFC233, css: '#FFC233', width: 2.5 },
  I2C:   { hex: 0x5E8BFF, css: '#5E8BFF', width: 1.8 },
  PCIE:  { hex: 0xD29CFF, css: '#D29CFF', width: 2.2 },
  SATA:  { hex: 0xFF8A3D, css: '#FF8A3D', width: 2.0 },
  USB:   { hex: 0x34D399, css: '#34D399', width: 1.6 },
};
```

### 3.2 你的节点数据需要带 `busConnections`

```ts
// 例：主板既是 I2C 主控，又挂在电源线上
{
  id: 'base_board',
  grid: { x: 1, y: 2, z: 0 },          // 位置（见 3.3）
  size: { w: 14, d: 14, h: 0.3 },      // 尺寸（端点引出位置要用）
  busConnections: [
    { busId: 'i2c_main',  busType: 'I2C',   role: 'master', connectorPos: 'top'    },
    { busId: 'pwr_main',  busType: 'POWER', role: 'slave',  connectorPos: 'right'  },
    { busId: 'pcie_main', busType: 'PCIE',  role: 'master', connectorPos: 'bottom' },
  ],
}
```

> ⚠️ **连线数据是"声明"出来的，不是从外部文件导入的。** 本项目这份连接关系
> 是按"真实 2U 服务器架构惯例"手写的演示数据（BMC 走 I2C 管理总线、PCIe root
> 在主板…）。**搬到新项目时，这一层是你必须自己填的**——填法见 3.4。

### 3.3 坐标契约：`grid` + `size` → 世界坐标

走线只认"节点的世界坐标"。本项目用一层 `grid → world` 映射（可整组居中）：

```ts
const g2wx = (gx: number) => gx + CX;   // CX/CZ = 让整个场景居中的偏移
const g2wy = (gz: number) => gz;        // 注意：grid.z 是"高度"
const g2wz = (gy: number) => gy + CZ;   // grid.y 是"深度"
```

> 如果你的项目里节点已经是世界坐标，**可以删掉这层映射**，让 `getConnectorPt3`
> 直接用节点 position。

### 3.4 聚合函数（直接抄）

```ts
export function getComponentsOnBus(busId: string): HardwareComponent[] {
  return HARDWARE_COMPONENTS.filter(c => c.busConnections.some(b => b.busId === busId));
}
```

**填连接关系的心智模型**：每条总线选 1 个 `master`（起点），其余都是 `slave`（终点）。
走线引擎自动从 master 向每个 slave 拉一根线。所以你只要回答两个问题：
1. 这条线的"中心/源头"是谁 → 设成 master；
2. 还有谁挂在这条线上 → 设成 slave。

## 4. ② 走线层 —— 分层 + 直角折线

```ts
// 每类总线一个高度层，不同协议的线永不在同一高度交叉（关键可读性技巧）
const WIRE_LAYER_Y = 3.4;
const BUS_LAYER_Y: Record<string, number> = {
  POWER: 2.9, I2C: 3.35, PCIE: 3.8, SATA: 4.25, USB: 4.7,
};

function computeRoute3(
  from: THREE.Vector3, to: THREE.Vector3,
  fromId: string, toId: string, idx: number,
  layerY: number = WIRE_LAYER_Y,
): THREE.Vector3[] {
  const Y = layerY;
  const zOff = idx * 0.14;          // 同条线上多个 slave 之间的横向错位，避免重叠
  const fx = from.x, fz = from.z + zOff;
  const tx = to.x,   tz = to.z   + zOff;
  const pts: THREE.Vector3[] = [];
  const push = (p: THREE.Vector3) => { if (!pts.length || p.distanceTo(pts[pts.length-1]) > 0.02) pts.push(p); };
  push(from.clone());
  push(new THREE.Vector3(fx, Y, from.z));                              // 抬到层高
  if (Math.abs(from.z - fz) > 0.02) push(new THREE.Vector3(fx, Y, fz));
  if (Math.abs(fx - tx)     > 0.04) push(new THREE.Vector3(tx, Y, fz)); // 走 X
  if (Math.abs(fz - tz)     > 0.04) push(new THREE.Vector3(tx, Y, tz)); // 走 Z
  if (Math.abs(tz - to.z)   > 0.02) push(new THREE.Vector3(tx, Y, to.z));
  push(to.clone());                                                    // 落到接口
  return pts;
}
```

`roundedCurve3(points, radius)` 把这串折点转成带圆角的 `CurvePath`（折角处插
`QuadraticBezierCurve3`）——直接抄，无需改。

`getConnectorPt3(comp, busId)` 按 `connectorPos`（top/bottom/left/right）算出线从
节点哪个面、哪个世界坐标引出——直接抄，只依赖 `grid/size` 和 `g2w*`。

## 5. ③④ 几何 + 样式层 —— `BusRouteLine`

一根线 = **圆管管体（ShaderMaterial）** + **两端 connector 接点**。核心：

```ts
// 管体：沿圆角曲线挤出的圆管
const tubeR = isTrunk ? 0.05 : 0.038;
const tubeGeo = new THREE.TubeGeometry(curvePath, divisions, tubeR, 7, false);

// 材质：实色 + 流动彗星高亮（uActive 时）
const mat = new THREE.ShaderMaterial({
  vertexShader: DASH_VERT, fragmentShader: DASH_FRAG,
  uniforms: {
    uColor:    { value: new THREE.Color(effColor) },
    uTotalLen: { value: totalLen },
    uOffset:   { value: 0 },      // useFrame 里递减 → 彗星向前流动
    uOpacity:  { value: alpha },  // useFrame 里缓动到目标透明度
    uActive:   { value: isActive ? 1 : 0 },
  },
  transparent: true, depthWrite: false, side: THREE.DoubleSide,
});
```

**两端接点**（搬过去一眼能看出"从哪连到哪"的关键）：

```
竖直接线柱(锥形 cylinder) + 底部色环(ringGeometry) + 白色中心点(circle)
透明度恒 ≥ 0.85 —— 线身即使 20% 半透，端点也始终醒目
```

着色器（直接抄，glsl 很短）：

```glsl
// DASH_VERT：把曲线弧长写进 vArcLen
varying float vArcLen;
void main(){ vArcLen = uv.x; gl_Position = projectionMatrix*modelViewMatrix*vec4(position,1.0); }

// DASH_FRAG：实色 + 一段沿线移动的白色亮带（彗星）
uniform vec3 uColor; uniform float uTotalLen,uOffset,uOpacity,uActive; varying float vArcLen;
void main(){
  vec3 col = uColor;
  float pos  = fract(vArcLen*uTotalLen*0.45 - uOffset);
  float band = smoothstep(0.0,0.03,pos)*(1.0-smoothstep(0.03,0.13,pos));
  col = mix(col, vec3(1.0), band*uActive*0.55);
  gl_FragColor = vec4(col, uOpacity);
}
```

每帧驱动（`useFrame`）：`uOffset` 递减让彗星流动；`uOpacity` 向目标缓动让 hover
"渐变变实"。错误态可叠加呼吸闪烁。

## 6. ⑤ 交互层

### 6.1 逐根 hover（每根线自己一个 state）
`BusRouteLine` 内部各持一个 `hovered`，`onPointerOver` 里 `stopPropagation`，
保证一次只亮一根（而不是整组一起亮）。`effAlpha = hovered ? 1.0 : alpha`。

### 6.2 透明度优先级（`BusLines` 里算 `baseAlpha`）
选中本线 1.0 > 选中的组件相连 0.95 > 有组件被选但不相关 0.05 > 默认 0.20 > 兜底 0.10。

### 6.3 分类显隐开关
```ts
// simStore
hiddenBusTypes: {} as Record<string, boolean>,
toggleBusType: (type) => set(s => ({ hiddenBusTypes: { ...s.hiddenBusTypes, [type]: !s.hiddenBusTypes[type] } })),

// 渲染处按类型过滤
busEntries.filter(({ bus }) => !hiddenBusTypes[bus.type]).map(...)

// 图例 UI：遍历 BUS_COLORS 生成 chip，颜色直接取 BUS_COLORS[type].css
```

## 7. 移植 Checklist（照着做）

1. **装依赖**：`three`、`@react-three/fiber`（已有 R3F 场景可跳过）。
2. **抄类型+颜色**：`BusType`/`BusConnection`/`BusDef`/`BUS_COLORS`/`BUS_REGISTRY`。
   改 `BusType` 联合类型和 `BUS_COLORS` 成你的线种类。
3. **给节点加 `busConnections`**：按 §3.4 的 master/slave 心智模型填。
4. **抄走线+渲染**：`roundedCurve3`、`computeRoute3`、`BUS_LAYER_Y`、
   `getConnectorPt3`、`BusRouteLine`、`BusLines`、两个 shader。
5. **接坐标**：让 `getConnectorPt3` 能拿到节点世界坐标（用你的 `grid→world`
   或直接用 position）。
6. **挂渲染**：场景里 `busEntries.map(e => <BusLines bus={e.bus} members={e.members}/>)`。
7. **（可选）开关**：抄 `hiddenBusTypes`/`toggleBusType` + `BusTypeSwitches` 图例。

## 9. 已知边界 / 注意

- **接点用 `<sprite>` 不要用 drei `<Html>`**：早期端点/徽标用 `<Html>`，每帧强制
  DOM reflow，DOM 一重就把主线程打满卡死。已全部换成 three 原生几何 / sprite。
- **材质统一遍历要排除线**：如果你的场景有"每帧 traverse 统一所有 mesh 材质"
  的逻辑，记得给线和接点打 `userData.keepMaterial` 跳过，否则彩色线会被刷成灰。
- **连线是示意数据**：见 §3.2，搬过去后连接关系要自己按真实拓扑填。
